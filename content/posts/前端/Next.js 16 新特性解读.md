Next.js 16 在 2026 年初正式发布，这次更新带来了不少让人眼前一亮的特性。最值得关注的是 Turbopack 的全面稳定化——开发服务器的启动速度和热更新都提升了一个数量级，大型项目终于不用在启动时去泡杯咖啡了。

另一个重磅更新是 Server Components 的进一步优化。现在你可以更细粒度地控制组件的渲染策略，Partial Prerendering 也进入了稳定阶段。这意味着你可以在同一个页面中混合使用静态内容、动态内容和流式渲染，而不用在构建时做任何妥协。

对于我个人的项目来说，最实用的是新的缓存 API。旧的 `cache()` 函数终于被替换成了更直观的 `unstable_cache` 替代方案，缓存失效策略也更清晰了。配合 React 19 的 `use()` hook，数据获取的体验比以前丝滑了不少。

不过迁移过程中也有一些坑。比如 middleware 的行为有 breaking change——现在 `redirect()` 不再抛出错误而是返回一个特殊对象，需要显式处理。如果你的项目里自定义了 middleware 逻辑，升级前最好先看一下官方的迁移指南。
